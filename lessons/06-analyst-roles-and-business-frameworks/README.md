BA 的价值不只是产出报表或模型，而是先判断自己面对的是运营执行、业绩监控、问题探索、因果衡量、参数优化还是数据治理问题，再用合适的业务框架把模糊决策拆成可验证的指标、假设和行动方案。

## 核心概念索引

- 运营分析师 (Operations Analyst)：把已知的运营需求转化为数据提取、报表和自动化流程。
- 业绩分析师 (Performance Analyst)：围绕既定 KPI 复盘表现、解释偏差并预测风险。
- 探索型分析 (Discovery)：从数据中发现尚未被定义的问题、机会和研究方向。
- 优化分析师 (Optimization Analyst)：在目标与约束已明确时，寻找最优参数或资源配置。
- 衡量分析师 (Measurement Analyst)：用实验和因果推断衡量业务改动的真实增量影响。
- 数据完整性分析师 (Integrity Analyst)：整合原始数据、统一口径并产出可信的标准指标。
- 盈利能力框架 (Profitability Framework)：从利润、收入、成本及其驱动因素逐层拆解问题。
- 4C 框架 (Customer, Product, Company, Competition)：从客户、产品、公司和竞争四个维度分析商业情境。
- 产能 (Capacity)：企业在一定时间内能够生产、交付或服务的最大能力。
- 单位经济模型 (Unit Economics)：判断每位客户、每笔订单或每单位业务是否有可持续经济价值。
- 获客成本 (Customer Acquisition Cost, CAC)：获得一位新客户的平均成本。
- 客户终身价值 (Lifetime Value, LTV)：客户在整个关系周期内带来的预期价值。
- 盈亏平衡点 (Break-even Point)：收入刚好覆盖成本时所需的销量、客户数或时间点。
- 回收期 (Payback Period)：收回初始投资所需的时间。

## 1. 为什么岗位名称会重叠

现实公司中，Analyst、Business Analyst、Product Analyst、Operations Analyst 等岗位名称并没有绝对统一的定义。小公司常由一个人覆盖多种工作；大公司才会按职能进一步拆分。

因此，判断一个分析岗位时，不应只问“它会不会分析原因”，而要看：

```text
问题从哪里开始？
最终交付给业务方什么？
```

例如“新买家数下降”可以被不同角色处理：

| 角色 | 主要问题 | 典型交付 |
| --- | --- | --- |
| Operations Analyst | 如何让团队更快发现并处理问题？ | 自动更新 KPI 看板、异常名单、告警流程 |
| Performance Analyst | 本周为什么未达标？月底能否追回？ | 业绩复盘、偏差解释、预测与风险提示 |
| Discovery Analyst | 新买家持续下降的真正机制是什么？ | 用户/渠道研究、问题定义、待验证假设 |
| Measurement Analyst | 某项挽留措施真的带来增量了吗？ | A/B Test、因果效果评估 |
| Optimization Analyst | 预算如何分配才能最大化新增买家？ | 最优预算/渠道/参数方案 |
| Integrity Analyst | 各部门看到的新买家数是否口径一致？ | 标准指标、ETL 流程、数据质量规则 |

## 2. 六类分析师的工作边界

### 2.1 运营分析师 (Operations Analyst)

运营分析以已知需求为起点，目标是让日常运营或项目执行更有效率。

常见工作：

- 生成应接收某类营销信息的买家名单；
- 建立并维护按品类展示畅销 SKU 的 Dashboard；
- 自动化重复的数据提取、清洗和分发流程。

所需能力包括数据提取 (Data Extraction)、数据处理 (Data Manipulation) 和自动化 (Automation)。

可以这样理解：

```text
业务已经知道要做什么 -> 运营分析让这件事做得更快、更稳定、更可规模化。
```

### 2.2 业绩分析师 (Performance Analyst)

业绩分析围绕已定义的指标和目标工作。它不仅“报数”，还要解释趋势、偏差和未来风险。

典型问题：

- 本周/本月实际表现与目标差多少？
- 差异来自哪个地区、渠道、产品或客户群？
- 当前趋势持续下去，季度目标能否完成？

常见交付包括 Weekly Business Review、KPI Dashboard、Forecast、风险提示和业绩深挖 (Deep Dive)。

### 2.3 探索型分析 (Discovery / Product / Business Analyst)

探索型分析从“我们还不知道真正的问题在哪里”出发。它通过用户行为、产品流程和业务生态寻找值得行动的洞察。

例如：

- 分析用户在购买流程中的行为，判断是否需要改版；
- 找出活跃买家流失 (Churn) 的主要触发因素，再决定是否启动项目。

它需要较强的产品与业务理解，也经常使用数据提取、统计和计量方法 (Econometrics)。

### 2.4 优化分析师 (Optimization Analyst)

优化问题的特点是目标与约束已较明确，但参数还没有最佳答案。

例如：

- 卖家手续费设多少，才能平衡平台收入、销量和卖家满意度？
- 广告如何配置，才能提高收入，同时不超过自然流量被侵蚀的阈值 (Cannibalization Threshold)？

常见场景包括 Pricing、广告投放、库存分配、物流路径与资源调度。它通常需要优化建模 (Optimization Modeling) 和决策算法 (Decision-making Algorithms)。

### 2.5 衡量分析师 (Measurement Analyst)

衡量分析师关注的是业务改动的真实增量 (Incrementality)：

> 如果没有这次改动，结果本来会怎样？实际结果与这个反事实 (Counterfactual) 的差，才是改动带来的增量效果。

例如价格调整后收入上升，并不一定完全由价格造成，也可能同时受旺季、竞争对手缺货或其他营销活动影响。

常用方法：

- A/B Test；
- 实验设计 (Experimentation)；
- 情景模拟 (What-if Scenario)；
- 计量经济学和因果推断 (Causal Inference)。

### 2.6 数据完整性分析师 (Integrity Analyst)

Integrity Analyst 的职责是把分散的原始数据整合为可复用、定义一致的标准指标。

例如 GMV (Gross Merchandise Value) 的口径必须说明：是否包含取消订单、退款如何处理、跨币种如何换算、不同团队是否使用同一个订单状态定义。

关键能力：

- ETL (Extract, Transform, Load)；
- 数据建模与编码；
- 指标定义、数据质量与治理。

课件中这页重复出现了“价格调整影响”和“A/B Test”的例子；它们更符合 Measurement Analyst，而不是 Integrity Analyst。Integrity 更合适的例子是网站流量标准化、GMV 口径统一、订单和用户数据清洗。

## 3. 利润与收入问题树 (Profitability Framework)

当业务提出“利润下降了”或“收入没有达标”时，不应直接猜原因，而应先按业务公式拆解。

```text
Profit = Revenue - Cost
Revenue = Revenue per Unit x Units Sold
Cost = Fixed Cost + Variable Cost
```

因此，利润变差可能来自：

1. 每单位收入/价格下降；
2. 销量下降；
3. 单位变动成本上升；
4. 固定成本上升；
5. 多个因素共同发生。

这个框架的用途是把模糊问题变成一系列可取数、可验证的子问题，而不是为了画出复杂的树状图。

## 4. 案例：电商平台收入下降 30%

案例背景：平台上的中国卖家向美国消费者销售电子产品，平台按销售额抽取交易手续费。2 月平台收入下降 30%。

已知信息：

- GMV 下降 30%；
- 支付给卖家的金额下降 30%；
- 交易笔数下降 30%；
- 总用户数基本持平；
- 买家数下降 10%；
- 活跃卖家数下降 30%；
- 同期正好处于中国春节假期。

### 4.1 先区分指标口径

总用户数、买家数和卖家数通常不是同一个口径：

- 总用户数可能包括注册账号、访问用户或总体活跃用户；
- 买家数是当期实际完成购买的用户；
- 卖家数是当期实际在售或成交的卖家。

春节期间，卖家可能只是暂时不营业、没有上架或没有成交，账号仍然存在；用户也可能仍在浏览而没有购买。因此总用户数持平，与买家/卖家下降并不矛盾。

### 4.2 初步诊断逻辑

```text
平台手续费收入 = Take Rate x GMV
GMV = 交易笔数 x 平均订单金额
```

交易笔数和活跃卖家数同时下降，买家下降幅度较小，初步假设是：

> 春节导致卖家侧供给暂时收缩，可售商品或可完成订单减少，从而使交易笔数、GMV 和平台手续费收入同步下降。

这还是待验证假设，不应直接写成因果结论。

### 4.3 应如何验证

1. 核实 GMV、交易、活跃卖家的口径和数据质量。
2. 对比春节前后及去年同期，判断是否为季节性模式。
3. 检查卖家的上架、库存、在线状态、发货时效和取消率。
4. 分解 GMV，确认主因是交易笔数还是平均订单金额。
5. 按卖家国家、品类、新老卖家进一步拆分，确认冲击是否集中在中国卖家与相关品类。

## 5. 案例：CPC 广告收入预测失准

案例背景：全球电商平台向卖家销售商品广告，收费模式是 CPC (Cost Per Click)。广告上线后，实际广告收入只有上线前预期的 10%。

### 5.1 用收入公式拆解

```text
广告收入 = 点击次数 x 每次点击价格
         = Impression x CTR x CPC (或平均 Bid)
```

对应盈利能力框架：

| 通用收入框架 | 广告业务中的含义 |
| --- | --- |
| Units Sold | 被收费的点击次数 (Clicks) |
| Revenue per Unit | 每次点击价格 (CPC) |
| Revenue | 广告收入 |

### 5.2 预测为什么可能出错

要逐项检查预测假设：

1. **Impression 高估**：实际访问流量、可投广告位或可展示机会少于预期。
2. **CTR 高估**：用户看到广告但不点击，可能是相关性、位置、创意或需求不匹配。
3. **CPC / Bid 高估**：卖家不愿出高价，竞价竞争不足。
4. **广告填充率或卖家采用率被忽略**：即使有流量，也未必有足够卖家购买广告来填满广告位。

分析原则：先验证公式中的关键假设，再决定是增加流量、改善广告质量、调整定价还是提高卖家采用率。

## 6. 商业情境框架 (Business Situation Framework / 4C)

当问题是“要不要进入新市场”“如何增长”“是否推出新产品”“如何扭转下滑”时，可以用 4C 来防止分析范围遗漏。

```text
Customer -> Product -> Company -> Competition
```

### 6.1 Customer（客户）

要回答：谁会买、市场是否足够大、需求是否增长。

- 识别客群 (Segments)；
- 看每个客群的规模、增长率和市场占比；
- 将当前指标与历史指标比较，识别趋势；
- 理解客户需求、使用场景和支付意愿。

### 6.2 Product（产品）

要回答：产品为何值得被买。

- 明确产品价值和用户收益；
- 判断产品是否同质化 (Commodity) 或有差异化 (Differentiation)；
- 找替代品 (Substitutes)；
- 找互补品 (Complements)，评估能否借助相关品类增长。

### 6.3 Company（公司）

要回答：公司是否具备把计划做成的能力。

- 能力、资源和专业性；
- 分销/交付渠道；
- 成本结构；
- 组织、技术、供应链、合规或服务能力是否成为限制。

### 6.4 Competition（竞争）

要回答：进入后是否有机会赢，或能否守住优势。

- 竞争集中度和市场结构：垄断 (Monopoly)、寡头 (Oligopoly) 或分散竞争；
- 主要对手和市场份额；
- 进入壁垒 (Barriers to Entry)：品牌、技术、渠道、规模、监管和网络效应等。

## 7. 新市场进入如何衡量成功

4C 框架在进入新市场后，应落到可监控的指标上。

| 维度 | 关键指标 | 说明 |
| --- | --- | --- |
| Customer | 新客户、下载/注册/访问、留存率、客户价值 | 不只看尝试人数，更看客户是否留下。 |
| Product | 每客户收入、CAC、流量、跳出率 | 看产品匹配度和获客效率。 |
| Company | 利润、收入、成本、品牌认知、机会成本 | 判断投资是否值得持续。 |
| Competition | 市场份额、增长率 | 反映长期竞争位置。 |

早期市场进入不应只用下载量或注册量判断成功。更核心的判断链是：

```text
获客 -> 激活/转化 -> 留存 -> LTV
                         |
                         v
                     LTV 是否大于 CAC？
```

若客户留下来并创造的长期价值能覆盖获取成本，市场进入才更可能可持续。市场份额通常是更滞后的结果指标。

## 8. 产能变更框架 (Capacity Change Framework)

公司考虑建新工厂、缩减产能、收购直接竞争对手，或在服务业增加人力与服务能力时，可以用以下三部分评估。

### 8.1 Demand（需求）

- 市场总体增长是否可持续？
- 公司市场份额增长是否可持续？
- 增长来自哪些需求细分市场？
- 哪些细分市场规模最大、增速最快？

课程中的 leverage points 指应优先研究“规模大且增速高”的细分市场，因为它们更可能决定未来大部分需求。

### 8.2 Supply（供给）

- 行业和各细分市场的现有供给有多少？
- 竞争者是否也会扩产？
- 新增供给是否会压低价格？
- 新产能能否引入技术、提高生产率、降低边际成本？

需求增长不必然等于应该扩产。若全行业同步扩产，供过于求可能压低价格，最终利润并不增加。

### 8.3 Cost of Expansion（扩张成本）

需同时考虑：

- 设备、场地、人力、管理和建设时间等真实成本；
- 回收期 (Payback Period)；
- 盈亏平衡点 (Break-even Point)；
- 机会成本：同一笔资源投向新产品、营销、收购或其他市场，是否更有价值。

简化判断：

```text
新增利润 = 新增收入 - 新增变动成本 - 新增固定成本
```

### 8.4 Alternatives（替代方案）

不一定要自己新建产能。还可比较：

- Outsource（外包）；
- Lease（租赁）；
- Sub-contract（分包）；
- 收购竞争对手；
- 通过流程或技术提高现有产能利用率。

## 9. 案例：是否在欧洲新建客服中心

客服中心的“产能”是一定时间内接听和解决电话/服务请求的能力，而不是工厂产量。

### 9.1 Demand：客户支持需求与业务损失

先问是否存在持续增长的服务需求，以及问题是否已经伤害业务：

- 来电量是否增长？
- 平均等待时间是否变长？
- 放弃接听率 (Abandonment Rate) 是否升高？
- 客服满意度 (CSAT) 是否下降？
- 这些问题是否导致流失、退款、投诉或收入损失？

还要按国家、语言、时段、产品和客户类型拆分。欧洲整体需求上涨，不代表每种语言或地区都需要单独新建中心。

### 9.2 Supply：量化需要增加多少服务能力

不能只说“客服不够”，要量化缺口：

- 每天额外需要接听多少电话；
- 目标等待时间是多少；
- 需要多少客服和什么语言能力；
- 现有团队的人均处理量、平均处理时长 (AHT) 与首次解决率 (FCR) 如何。

还要先判断瓶颈是否真的来自人手不足，还是来自流程、排班、系统、知识库或问题本身。

### 9.3 成本、收益与替代方案

新中心的成本包括场地、设备、招聘、培训、管理和启动时间。收益不是“多接几通电话”本身，而是：

```text
更短等待时间/更快解决问题
-> 更高满意度和留存
-> 更少退款、投诉与流失
-> 被保护或新增的收入
```

还应比较：提升现有团队效率、在其他地点设置中心、使用多语言外包服务等方案。

## 10. BA 视角总结

这节课最值得形成的工作顺序是：

```text
1. 先识别问题类型和分析角色
2. 用业务公式或框架拆问题，而不是凭直觉归因
3. 明确每个子问题的指标、数据口径、比较对象和时间范围
4. 区分现象、假设、证据和因果结论
5. 比较多种方案的增量收益、成本、风险和机会成本
6. 把结论转化为可执行的决策、实验或运营动作
```

对于 BA，框架不是为了套模板，而是为了确保复杂决策中不会漏掉关键问题，并让分析过程可复查、可沟通、可落地。

## 12. 术语表

| 术语 | 含义 |
| --- | --- |
| Operations Analyst | 以运营效率、报表、数据提取和自动化为重点的分析角色。 |
| Performance Analyst | 围绕 KPI、目标差距、趋势、预测和风险工作的分析角色。 |
| Discovery | 从数据中发现未知问题、机会和待验证假设的分析方式。 |
| Optimization | 在目标与约束明确时寻找最优配置的分析问题。 |
| Incrementality | 某项干预相对“不做干预”所真正带来的额外效果。 |
| Integrity | 数据定义一致、质量可靠、可被不同团队复用的状态。 |
| Profitability Framework | 从利润、收入、成本及其驱动因素拆解商业问题的框架。 |
| Take Rate | 平台从交易金额中抽取的比例，例如佣金或手续费率。 |
| GMV | Gross Merchandise Value，平台上完成或产生的商品交易总额，具体口径需说明。 |
| CPC | Cost Per Click，按广告点击次数收费的模式或每次点击价格。 |
| 4C Framework | Customer、Product、Company、Competition 的商业分析框架。 |
| CAC | Customer Acquisition Cost，获得一位新客户的成本。 |
| LTV | Lifetime Value，客户在生命周期内带来的预期价值。 |
| Capacity | 企业在一定时间内可生产、交付或服务的最大能力。 |
| Marginal Cost | 每额外增加一单位产出所增加的成本。 |
| Opportunity Cost | 选择某一方案而放弃其他方案可能带来的价值。 |
| Payback Period | 收回初始投资所需时间。 |
| Break-even Point | 收入刚好覆盖成本时的销量、客户数或时间点。 |
